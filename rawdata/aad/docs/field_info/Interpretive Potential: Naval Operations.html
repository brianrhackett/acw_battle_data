NARA Notes:  The "events" database was converted from a fixed length file to a csv (comma delimited) file during NARA processing. The "IPOTENTIAL" field was a twenty two-character field in the fixed length file with eighteen possible single character codes. The Commission allowed the field to contain multiple interpretive codes. However, many records had only one or no interpretive codes. In order to prevent technical problems, the original twenty two-character "IPOTENTIAL" field was split up into eighteen single character fields for each possible interpretive code. The one character code used for each one of the eighteen "IPOTENTIAL" fields corresponds to a "yes" or "no" value.
